# Термінологічний глосарій

Для фахового розуміння процесів взаємодії з периферійними пристроями в Linux-системах необхідно оперувати наступними поняттями:

* **Монтування (Mounting)** — логічна операція, що створює асоціативний зв'язок між фізичним розділом накопичувача та конкретним каталогом у дереві файлової системи.
* **Блочний пристрій (Block Device)** — абстракція ядра для обладнання, що дозволяє читати та записувати дані блоками фіксованого розміру (наприклад, `/dev/sdb`).
* **Точка монтування (Mount Point)** — звичайна директорія (найчастіше в `/mnt` або `/media`), яка після процедури монтування стає "входом" до вмісту зовнішнього носія.
* **VFS (Virtual File System)** — рівень ядра Linux, який дозволяє користувачеві працювати з різними файловими системами (FAT32, NTFS, ext4) через єдиний стандартний інтерфейс.
* **Udev** — менеджер пристроїв, який динамічно створює файли пристроїв у каталозі `/dev` при підключенні флеш-накопичувача.

# 1. Теоретичний базис та архітектурні особливості

## 1.1 Алгоритм ініціалізації пристрою ядром Linux 
Коли ми вставляємо флеш-накопичувач у порт, система проходить складний шлях ідентифікації:

1.  **Детекція:** Контролер USB фіксує зміну напруги, і ядро отримує сигнал про новий пристрій.
2.  **Ідентифікація:** Драйвер USB-storage зчитує дескриптори пристрою (ID виробника, об'єм, тип таблиці розділів).
3.  **Призначення імені:** Система присвоює накопичувачу літерний індекс. Якщо основний диск — `sda`, то флешка зазвичай стає `sdb`, а її перший розділ — `sdb1`.
4.  **Автомонтування (GUI):** В сучасних десктоп-оточеннях (наприклад, GNOME) служба `udisks` автоматично монтує розділ у `/media/$USER/label`.

## 1.2 Концептуальна різниця: Linux VS Windows
Взаємодія з носіями в цих двох системах базується на діаметрально протилежних парадигмах:

| Критерій порівняння | Парадигма Linux  | Парадигма Windows  |
| :--- | :--- | :--- |
| **Логіка ієрархії** | **Єдине дерево.** Немає поняття "дисків", є лише папки. | **Розділені корені.** Кожен пристрій — це нова літера (C:, D:). |
| **Філософія пристроїв** | **"Все є файлом".** Флешка представлена як файл у `/dev`. | Пристрої як об'єкти з окремими інтерфейсами керування. |
| **Процес підключення** | Вимагає явної вказівки точки входу (Mount Point). | Автоматичне призначення літери диска системою. |
| **Контроль доступу** | Права доступу інтегровані в загальну систему POSIX. | Використання списків контролю доступу (ACL). |

# 2. Практичний протокол виконання роботи

У ході виконання завдання ми розглянули два сценарії взаємодії: через графічну оболонку для пересічного користувача та через командний рядок для системного адміністратора.

## 2.1 Конфігурація віртуального середовища (VirtualBox)
Оскільки робота проводилася у віртуальній машині, першим критичним етапом було прокидання (passthrough) фізичного порту:
* Ми підключили флеш-карту до фізичного USB-порту хост-машини.
* У налаштуваннях VirtualBox перейшли до розділу **USB** та додали фільтр для нашого пристрою.
* Це дозволило гостьовій ОС Ubuntu перехопити керування драйвером у Windows-хоста, після чого флешка зникла з "Провідника" Windows і з'явилася в терміналі Linux.

## 2.2 Взаємодія через графічний інтерфейс (GUI)
Цей метод є найбільш наочним:
1.  **Детекція:** Одразу після прокидання пристрою на панелі завдань Ubuntu з'явилася піктограма накопичувача.
2.  **Навігація:** Ми відкрили файловий менеджер Nautilus. Флешка відобразилася у лівій панелі під своєю назвою.
3.  **Маніпуляція даними:** Ми виділили цільовий файл на флешці, скористалися контекстним меню для копіювання і перенесли його у папку `~/Documents`. 
4.  **Безпека:** Перед вийманням ми натиснули кнопку "Eject" біля назви пристрою. Це критично, оскільки Linux використовує кешування запису, і передчасне виймання може призвести до пошкодження файлової системи.

## 2.3 Професійна робота через CLI (Термінал)
Найважливіша частина роботи, де ми відтворили всі дії вручну за допомогою команд.

**Крок 1: Діагностика блочних пристроїв**
Виконавши команду `lsblk`, ми побачили дерево пристроїв. Флешка ідентифікувалася як `sdb` з розділом `sdb1` розміром 16 ГБ. Ми зафіксували шлях до пристрою: `/dev/sdb1`.

**Крок 2: Створення структури для підключення**
Оскільки ми не хотіли використовувати автоматичні шляхи, ми створили власну точку монтування:
`sudo mkdir -p /mnt/external_usb`
Це пуста директорія, яка тимчасово стане вітриною вмісту нашої флешки.

**Крок 3: Виконання монтування**
За допомогою команди `sudo mount /dev/sdb1 /mnt/external_usb` ми підключили пристрій. Важливо зауважити, що на цьому етапі ядро Linux перевірило тип файлової системи (FAT32) і підвантажило відповідний драйвер.

**Крок 4: Керування файлами через консоль**
Ми переконалися, що файли доступні, командою `ls -la /mnt/external_usb`. Далі виконали копіювання:
`cp /mnt/external_usb/report.pdf ~/Desktop/`
Це підтвердило успішне читання даних з носія.

**Крок 5: Розмонтування (Unmounting)**
Після завершення операцій ми звільнили точку монтування:
`sudo umount /mnt/external_usb`
Система скинула всі буфери запису на фізичний носій. Тільки після цього пристрій було безпечно вимкнено.
